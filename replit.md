# WinTube v2.0

A secure "Watch & Earn" platform where users earn points by watching YouTube Shorts and completing offers. Points can be withdrawn as cryptocurrency, bank transfers, or mobile credit.

## Tech Stack

- **Framework**: Hono (lightweight web framework for edge runtimes)
- **Language**: TypeScript
- **Runtime**: Cloudflare Workers / Pages (via Wrangler)
- **Database**: Cloudflare D1 (SQLite-based, local dev via Wrangler)
- **Build Tool**: Vite with @hono/vite-build

## Project Structure

- `src/index.tsx` — Main app entry point (frontend HTML + API routes)
- `src/routes/` — API route handlers (auth, videos, points, social)
- `src/middleware/` — Auth middleware (JWT/Session validation)
- `src/lib/` — Utilities (crypto, rate limiting, types)
- `migrations/` — D1 SQL schema migrations
- `public/` — Static assets (CSS)
- `dist/` — Built output (generated by `npm run build`)

## Development

The app runs via Wrangler Pages dev server:
```
npm run build && npx wrangler pages dev dist --d1=DB --local --ip 0.0.0.0 --port 5000
```

Database setup (first time or reset):
```
npx wrangler d1 migrations apply wintube-db --local
```

## Environment Variables / Secrets

- `YOUTUBE_API_KEY` — Required for video feed (Cloudflare secret / env var)
- `JWT_SECRET` — For session token signing (optional, defaults to random)

## Notes

- The YouTube API key must be set as a Cloudflare secret (or local env var) for the video feed to work
- Local D1 state is stored in `.wrangler/state/v3/d1/`
- Port 5000 is used for both dev and production (Replit webview)
